    第3题：浏览器缓存 ETag 里的值是怎么生成的

    ETag 是 Entity Tag 的缩写，实体标签 的意思

    ETag 是Web的一种缓存机制，用来表示某个资源是否已经被修改了，是否需要重新下载。没有被修改的话，可以返回 304 HTTP Code。

    HTML 标准里面没有ETag具体的生成算法，所以需要自己写或者用 Nginx、Apache 等服务的生成算法。

    大体是 last-modified-time 和 content-length 的16进制组合。

    第4题：null 和 undefined 有什么区别？

    相同点

    大写的 Null 和 Undefined 是 JavaScript 的基本数据类型，都只有一个值，对应的是 null 和 undefined。

    都表示一个 无效的值，所以不能访问属性，会报错 null``.toString(); undefined.toString();。

    不同点

    Undefined 的字面意思就是：未定义的值 。这个值的语义是，希望表示一个变量最原始的状态，而非人为操作的结果 。最后一句是重点。

    这种原始状态会在以下 4 种场景中出现：

    1. 声明了一个变量，但没有赋值

    2. 访问对象上不存在的属性

    3. 函数定义了形参，但没有传递实参

    4. 使用 void 对表达式求值

    Null 的字面意思是：空值 。这个值的语义是，希望表示 一个对象被人为的重置为空对象，而非一个变量最原始的状态 。 在内存里的表示就是，栈中的变量没有指向堆中的内存对象，即：

    当一个对象被赋值了null 以后，原来的对象在内存中就处于游离状态，GC 会择机回收该对象并释放内存。因此，如果需要释放某个对象，就将变量设置为 null，即表示该对象已经被清空，目前无效状态。试想一下，如果此处把 null 换成 undefined 会不会感到别扭? 显然语义不通，其操作不能正确的表达其想要的行为。

    null 有属于自己的类型 Null，而不属于Object类型，typeof 之所以会判定为 Object 类型，是因为 JavaScript 数据类型在底层都是以二进制的形式表示的，二进制的前三位为 0 会被 typeof 判断为对象类型，而 null 的二进制位恰好都是 0 ，因此，null 被误判断为 Object 类型。

    000 - 对象，数据是对象的应用

    001 - 整型，数据是31位带符号整数

    010 - 双精度类型，数据是双精度数字

    100 - 字符串，数据是字符串

    110 - 布尔类型，数据是布尔值

    第8题：浏览器拿到 index.html 之后做了什么？

    • HTML代码转化成 DOM，并并行地获取外部资源（包括HTML中出现的JS、CSS、图片文件）
    • CSS文件下载完成，CSS代码转化成 CSSOM（CSS Object Model）
    • 结合DOM和CSSOM，生成一棵 渲染树（包含每个节点的视觉信息）
    • 生成布局（Layout）：将所有渲染树的所有节点进行平面合成，该步骤可以计算出元素在网页中的位置
    • 绘制（Paint）：将网页内容绘制在屏幕上

    小总结

    • 外部资源是在开头并行获取的，并非HTML解析到引用行再发请求的；
    • CSS文件加载的速度是影响首次Paint时间的因素，当CSS文件下载完成后才能进行CSSOM和渲染树的构建，然后才是Layout和Paint过程；
    • JS文件加载的速度不影响首次Paint时间，即使JS文件本身有修改DOM节点的操作，也只是在JS文件加载完成后再进行一次Layout和Paint；

    各类资源阻塞网页加载的情况

    阻塞分两种：

    1. 阻塞 HTML 解析
    2. 阻塞 渲染

    CSS 影响首次绘制时间，应该尽快加载，因此将其放在head标签处以保障能在解析完HTML后发出请求；

    JS 影响 HTML 解析 和 渲染，但不影响首屏，他影响的是 JS 下载完之后页面内容的解析与渲染；

    第9题：String.slice 和 String.substring 的区别

    都是用来提取字符串，都生成新的字符串

    相似点

    1. 如果 start 和 stop 相等，返回 空字符串
    2. 如果省略 stop，一直提取到字符串末尾
    3. 如果任意一个参数大于字符串的长度，则会用字符串的长度代替这个参数

    substring()的不同点

    1. 如果 start 大于 stop，则会交换两个参数的位置
    2. 如果任意一个参数是 负数 或者 NaN，则会当作 0 来处理

    slice() 的不同点

    1. 如果 start 大于 stop，则返回 空字符串
    2. 如果 start 是负数，会用 Math.max(0, 字符串长度+start) 来代替
    3. 如果 stop 是负数，会用 Math.max(0, 字符串长度+end) 来代替

    总结

    1. 如果字符串非空，除非 start 和 stop 相等，或者 从 stop 开始提取，否则 substring() 一定会返回字符，substring() 的容错率高一些
    2. slice() 处理负数会拿字符串长度相加，substring() 直接取 0

    第10题：如何描述XSS的危害

    如果一个恶意脚本在你的页面上被运行，它就有权访问这个页面的所有状态和执行该页面所能做的所有操作。

    它能与你的服务器进行交互，而你的服务器将不能区分正当请求和恶意请求。

    恶意脚本还能访问全局变对象，这使得它有权访问应用中除隐藏于闭包中的变量之外的所有数据。

    它可以访问document对象，这会使它有权访问用户所能看到的一切。

    它还给这个恶意脚本提供了与用户进行会话的能力。

    浏览器的地址栏和所有的反钓鱼程序会告诉用户这个会话是可靠的。

    document对象还给该恶意脚本授权访问网络，允许它取下载更多的恶意脚本，或者是在你的防火墙之内探测站点，或者是把它已经窃取的隐私内容发送给世界的任何一个服务器。
